Youbike 預測網站架設

動機與目標

動機
為了不要因為沒有腳踏車借導致上課遲到
喜歡逛展覽，但是每次找展覽或是活動，都發現想去玩的地方離自己太遠，或是明明附近有活動卻不知道
所以就想要做一個可以查看附近哪裡有活動進行中的網站
方便出去玩
 目標
網站上可以看到指定位置有什麼展覽以及活動
可以直接跳轉到活動報名網址
可以預測未來一小時以內的youbike站點狀態
讓人不用白跑一趟

抓取資料的網站

Link : YouBike2.0 臺北市公共自行車即時資訊
使用github workflow + google cloud 設置爬蟲
每兩分鐘從API抓取一次全台北市的youbike站點狀態
在雲端硬碟存成json檔案
 Link : 非池中藝術網
靜態網站
使用github workflow設置爬蟲，每日抓取一次網站資訊
再用google cloud api將抓下來的地址轉址成gps座標
呈現在地圖上
 Link : ACCUPASS 活動通
動態網站
同樣使用github workflow設置爬蟲，每日抓取一次網站資訊
再用google cloud api將抓下來的地址轉址成gps座標
呈現在地圖上

預測使用的模型

隨機森林模型 使用這個模型的原因:
1.預測速度快，適合放網站
2.要預測的時間很短
3.1596個youbike站點的一個月資料太多,跑時間模型跑不完
4.影響腳踏車變化的因素太多，隨機森林不用過多假設
 模型設定:
1.預測某個 YouBike 站點在未來5分鐘的可借車輛數量
2.分為假日與平日兩種情況
 模型架設:
1.用FastAPI建立YouBike可借車輛數預測API服務
2.Link : 預測API連結
